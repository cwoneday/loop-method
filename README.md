# LOOP — 작업 방법론 허브

> 완료는 선언이 아니라 **통과**의 결과다.

작업자와 채점자를 분리해, "완료했습니다"라는 선언이 아니라 **기준 통과**로 완료를 정의하는 작업 원칙(LOOP)을 정리한 정적 사이트입니다.

## 페이지

- **`index.html`** — 허브. 원리, 작업자↔채점자 분리, 채점 체크리스트, 완료 양식, 안전장치, 다중 에이전트로의 확장
- **`templates.html`** — 복사용 템플릿. 원칙 전문, 완료 양식, 채점 체크리스트, 모드 전환 선언, 프로젝트 `LOOP_*.md` 골격

## 로컬에서 보기

`index.html`을 브라우저로 열면 됩니다. 단, 템플릿 페이지의 복사 버튼은 보안 컨텍스트(https)에서 가장 안정적으로 동작합니다 — GitHub Pages 배포 후를 권장합니다.

## 배포 (GitHub Pages)

저장소 **Settings → Pages → Build and deployment → Source: Deploy from a branch → Branch: `main` / `(root)` → Save**.
1~2분 뒤 `https://<사용자명>.github.io/<저장소이름>/` 에서 열립니다.

이 저장소에는 Jekyll 처리를 끄기 위한 빈 `.nojekyll` 파일이 포함되어 있습니다.
